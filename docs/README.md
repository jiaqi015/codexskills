# Documentation Index

## Why This Structure

The documentation is split by lifecycle stage so contributors can find the right guidance quickly:

- **Getting Started**: onboarding and repository map
- **Authoring**: how to write high-quality skills
- **Operations**: ownership, governance, versioning
- **Templates**: copy-paste scaffolds
- **ADR**: key architectural decisions

- Skill maintainers own `docs/authoring/*` and skill-local docs
