# ADR 0001: Documentation Structure

## Status

Accepted

## Context

The repository needs a scalable documentation model for onboarding, skill authoring, and operations.

## Decision

Adopt lifecycle-based documentation sections:

- `getting-started`
- `authoring`
- `operations`
- `templates`
- `adr`

## Consequences

- Faster onboarding for new contributors
- Clear ownership boundaries
- Lower chance of missing process-critical documentation
