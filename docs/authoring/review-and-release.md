# Review and Release Flow

## PR Scope

One PR should cover one coherent skill change set.

## PR Must Include

1. What changed
2. Why the change is needed
3. How it was validated
4. Any migration impact

## Reviewer Focus

- Trigger correctness
- Determinism for fragile workflows
- Security and secret handling
- Documentation completeness

## Release Pattern

1. Merge PR
2. Tag release if needed
3. Update changelog/release notes if repository uses one
